Recipe Timer Orchestrator

A friendly mock workflow that tests parallel timers, resource constraints, user feedback, cancellation, and dynamic replanning.

What It Should Do

Coordinate a toy recipe with steps that can run in parallel, timers that must fire in order, limited equipment, and taste-test feedback that can add extra cooking time or cancel remaining steps.

Mocked Programs and Services

  • Recipe JSON fixture.
  • Fake clock for all timers.
  • Equipment locks for oven, pan, and bowl.
  • User feedback signals such as too_salty, needs_more_time, or serve_now.

Mermaid Diagram

flowchart TD
  A[Load recipe] --> B[Check ingredient inventory]
  B --> C{Missing optional ingredient?}
  C -- yes --> D[Apply substitution]
  C -- no --> E[Acquire equipment locks]
  D --> E
  E --> F[Start parallel prep timers]
  F --> G[Wait for all prep complete]
  G --> H[Start cook timer]
  H --> I[Ask user for taste signal]
  I --> J{Feedback}
  J -- needs_more_time --> H
  J -- cancel --> K[Release locks and stop]
  J -- serve_now --> L[Release locks and publish recipe log]

Example Inputs

{
  "recipeId": "mock-soup",
  "ingredients": { "water": 1, "salt": 1, "noodles": 1 },
  "availableEquipment": ["pot", "bowl"],
  "steps": [
    { "id": "boil", "durationSeconds": 5, "equipment": "pot" },
    { "id": "prep-bowl", "durationSeconds": 2, "equipment": "bowl" },
    { "id": "cook-noodles", "durationSeconds": 6, "after": ["boil"] }
  ],
  "feedbackSignals": ["needs_more_time", "serve_now"]
}

  "timelineArtifact": "memory://recipes/mock-soup.timeline.json"
}

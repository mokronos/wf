Text Redaction Review

A text-only workflow for testing classification, branch policies, human review, patch application, and versioned artifacts.

What It Should Do

Scan a short text fixture for sensitive tokens, redact low-risk matches automatically, request reviewer confirmation for ambiguous matches, apply reviewer patches, and publish both redacted text and a review log.

Mocked Programs and Services

  • String input instead of PDFs or document services.
  • Regex detector with deterministic confidence scores.
  • Reviewer decisions supplied as test signals.
  • Versioned text artifact store in memory.

Mermaid Diagram

flowchart TD
  A[Load text] --> B[Detect sensitive spans]
  B --> C[Split auto and review spans]
  C --> D[Apply automatic redactions]
  D --> E{Review spans exist?}
  E -- no --> I[Publish redacted text]
  E -- yes --> F[Create review task]
  F --> G[Wait for reviewer patch]
  G --> H{Patch valid?}
  H -- no --> F
  H -- yes --> I
  I --> J[Store version and audit log]

Example Inputs

{
  "documentId": "doc-redact-001",
  "text": "Call Ada at ada@example.test. Account token maybe ABC-123. Public code is DEMO-000.",
  "rules": [
    { "name": "email", "pattern": "[^ ]+@[^ ]+", "action": "auto_redact" },
    { "name": "token", "pattern": "[A-Z]{3}-[0-9]{3}", "action": "review" }
  ],
  "reviewSignals": [
    { "span": "ABC-123", "decision": "redact" },
    { "span": "DEMO-000", "decision": "keep" }
  ]
}

Example Outputs

{
  "documentId": "doc-redact-001",
  "status": "published",
  "redactedText": "Call Ada at [EMAIL]. Account token maybe [TOKEN]. Public code is DEMO-000.",
  "redactions": [
    { "span": "ada@example.test.", "rule": "email", "source": "auto" },
    { "span": "ABC-123", "rule": "token", "source": "reviewer" }
  ],
  "version": 2,
  "auditArtifact": "memory://redactions/doc-redact-001.audit.json"
}

Retryable Task Queue

A fake queue workflow for testing leases, retries, backoff, poison messages, and worker cancellation.

What It Should Do

Consume deterministic messages from an in-memory queue, process each message with configured failure scripts, retry transient failures with fake-clock backoff, send poison messages to a dead-letter queue, and stop cleanly when cancelled.

Mocked Programs and Services

  • In-memory queue with deterministic lease expiration.
  • Scripted processor returning ok, transient_error, or permanent_error.
  • Fake clock for backoff and lease timeout tests.
  • Dead-letter list as a local JSON artifact.

Mermaid Diagram

flowchart TD
  A[Start worker group] --> B[Lease next message]
  B --> C{Message available?}
  C -- no --> D[Wait for poll interval]
  D --> B
  C -- yes --> E[Run scripted processor]
  E --> F{Result}
  F -- ok --> G[Ack message]
  F -- transient --> H[Schedule retry with backoff]
  F -- permanent --> I[Dead-letter message]
  H --> J{Attempts exhausted?}
  J -- no --> B
  J -- yes --> I
  G --> K{Cancel signal?}
  I --> K
  K -- no --> B
  K -- yes --> L[Release leases and summarize]

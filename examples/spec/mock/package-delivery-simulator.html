Package Delivery Simulator

A fully fake fulfillment workflow for testing sagas, compensation, inventory reservations, and status subscriptions.

What It Should Do

Reserve toy inventory, pack a mock package, simulate carrier updates from a fixture, handle failed delivery attempts, refund or reship when needed, and notify subscribers of every state transition.

Mocked Programs and Services

  • In-memory inventory table.
  • Fake carrier update stream.
  • Fake notification sink that records messages.
  • Compensation functions for releasing stock and issuing mock refunds.

Mermaid Diagram

flowchart TD
  A[Create order] --> B[Reserve inventory]
  B --> C{Reserved?}
  C -- no --> D[Reject order]
  C -- yes --> E[Pack package]
  E --> F{Packing failed?}
  F -- yes --> G[Release inventory and notify failure]
  F -- no --> H[Create fake shipment]
  H --> I[Consume carrier events]
  I --> J{Delivered?}
  J -- yes --> K[Complete order]
  J -- no --> L{Attempts exhausted?}
  L -- no --> I
  L -- yes --> M[Choose refund or reship]
  M --> N[Run compensation or replacement saga]

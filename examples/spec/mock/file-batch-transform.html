File Batch Transform

A local file workflow using tiny text fixtures to test watchers, manifests, retries, quarantine, and resumability.

What It Should Do

Pretend a folder watcher found CSV files. Validate headers, normalize rows, calculate checksums, write output files, quarantine bad files, and resume correctly if the run is interrupted halfway through.

Mocked Programs and Services

  • Fixture directory such as fixtures/inbox/*.csv.
  • Fake file watcher event list instead of OS notifications.
  • Temporary directory or in-memory filesystem for transformed outputs.
  • Manifest JSON file for resume and idempotency checks.

Mermaid Diagram

flowchart TD
  A[Receive file events] --> B[Deduplicate paths]
  B --> C[For each file validate header]
  C --> D{Valid?}
  D -- no --> E[Move to quarantine and record reason]
  D -- yes --> F[Normalize rows]
  F --> G[Write output temp file]
  G --> H[Checksum output]
  H --> I[Atomic rename to done]
  E --> J[Update manifest]
  I --> J
  J --> K{More files?}
  K -- yes --> C
  K -- no --> L[Emit summary]

Example Inputs

{
  "runId": "file-run-001",
  "events": ["inbox/a.csv", "inbox/b.csv", "inbox/a.csv", "inbox/bad.csv"],
  "files": {
    "inbox/a.csv": "name,amount\nAda,10\nLinus,20\n",
    "inbox/b.csv": "name,amount\nGrace,30\n",
    "inbox/bad.csv": "wrong,columns\nx,y\n"
  },
  "resumeManifest": null
}

Example Outputs

{
  "runId": "file-run-001",
  "status": "completed",
  "processed": ["inbox/a.csv", "inbox/b.csv"],
  "quarantined": [{ "path": "inbox/bad.csv", "reason": "missing required headers: name, amount" }],
  "outputs": [
    { "path": "done/a.ndjson", "checksum": "sha256:mock-a" },
    { "path": "done/b.ndjson", "checksum": "sha256:mock-b" }
  ],
  "manifest": "memory://manifests/file-run-001.json"
}

Executable Fixture Contract

Fully specified node-by-node cases for this workflow live in fixtures/fully-specified-workflows.json under file-batch-transform. The tests assert dedupe, validation, normalization, checksums, summary output, and cache replay.

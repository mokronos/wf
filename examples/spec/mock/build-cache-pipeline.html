Build Cache Pipeline

A local-only CI/CD-shaped workflow with fake commands, dependency ordering, cache hits, flaky tests, and rollback.

What It Should Do

Run a miniature build graph with deterministic command results. Skip cached tasks, retry flaky tests, publish artifacts only after all dependencies pass, and roll back the publish marker if post-publish verification fails.

Mocked Programs and Services

  • Fake command runner that reads scripted results instead of executing shell commands.
  • In-memory cache keyed by task input hashes.
  • Artifact store represented by local JSON objects.
  • Post-publish verifier that can be scripted to fail.

Mermaid Diagram

flowchart TD
  A[Load build graph] --> B[Topologically sort tasks]
  B --> C{Task cache hit?}
  C -- yes --> D[Reuse artifact]
  C -- no --> E[Run fake command]
  E --> F{Command result}
  F -- pass --> G[Store task artifact]
  F -- flaky_fail --> H[Retry task]
  H --> E
  F -- fail --> I[Fail pipeline]
  D --> J{All dependencies complete?}
  G --> J
  J -- no --> C
  J -- yes --> K[Publish release marker]
  K --> L[Run post-publish verification]
  L --> M{Verified?}
  M -- yes --> N[Complete]
  M -- no --> O[Rollback marker and fail]

Example Inputs

{
  "buildId": "build-001",
  "tasks": [
    { "id": "compile", "deps": [], "hash": "h1", "script": ["pass"] },
    { "id": "unit-test", "deps": ["compile"], "hash": "h2", "script": ["flaky_fail", "pass"] },
    { "id": "package", "deps": ["unit-test"], "hash": "h3", "script": ["pass"] }
  ],
  "cache": { "h1": "artifact://compile-cached" },
  "postPublishScript": ["pass"]
}

    { "id": "package", "source": "executed", "attempts": 1 }
  ],
  "releaseMarker": "memory://releases/build-001",

}

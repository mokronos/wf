Arithmetic Approval Lab

A deterministic math workflow that still exercises approvals, correction loops, timers, audit logs, and idempotent finalization.

What It Should Do

Receive a batch of arithmetic problems, solve them, compare results to expected tolerances, route suspicious answers to a human reviewer, wait for corrections, and publish a signed score report. All inputs are local JSON and all human actions can be simulated with signals.

Mocked Programs and Services

  • Local JSON fixture loader for problem batches.
  • Fake human review inbox implemented as a test signal queue.
  • Fake clock for review deadlines and reminder timers.
  • In-memory artifact store for reports and audit entries.

Mermaid Diagram

flowchart TD
  A[Load problem batch] --> B[Fan out solve tasks]
  B --> C[Collect answers]
  C --> D{Any tolerance failures?}
  D -- no --> H[Generate score report]
  D -- yes --> E[Create human review tasks]
  E --> F{Review signal before deadline?}
  F -- no --> G[Mark unresolved and apply default penalty]
  F -- yes --> C
  G --> H
  H --> I[Persist report idempotently]
  I --> J[Emit completion event]

Example Inputs

{
  "batchId": "math-batch-001",
  "deadlineSeconds": 30,
  "problems": [
    { "id": "p1", "op": "add", "args": [12, 30], "expected": 42 },
    { "id": "p2", "op": "divide", "args": [22, 7], "expected": 3.142, "tolerance": 0.01 },
    { "id": "p3", "op": "sqrt", "args": [144], "expected": 12 }
  ],
  "reviewSignals": [
    { "problemId": "p2", "correctedAnswer": 3.142857, "reviewer": "test-human" }
  ]
}

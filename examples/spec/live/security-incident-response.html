Security Incident Response

Business workflow for triage, containment, evidence collection, customer communication, and postmortem closure.

What It Should Do

React to a high-confidence security alert, create an incident room, gather evidence, coordinate responders, run containment steps, decide whether notification obligations apply, and close with a postmortem and control improvements.

Programs and Services Involved

Detection
SIEM, EDR, cloud audit logs, identity provider logs, vulnerability scanner.
Coordination
PagerDuty, Slack or Teams, incident management system, ticket tracker, status page.
Governance
Legal hold storage, customer comms tool, data inventory, evidence bucket, postmortem docs.

Workflow Diagram

flowchart TD
  A[SIEM raises high-confidence alert] --> B[Create incident and page commander]
  B --> C[Open war room and evidence folder]
  C --> D[Collect account, host, network, and app context]
  D --> E{Confirmed incident?}
  E -- no --> F[Close as false positive with tuning task]
  E -- yes --> G{Severity}
  G -- sev3 --> H[Contain affected assets]
  G -- sev2 --> I[Contain and page security lead]
  G -- sev1 --> J[Contain, page execs, notify legal]
  H --> K[Forensic snapshot and timeline]
  I --> K
  J --> K
  K --> L{Customer data exposed?}
  L -- unknown --> M[Run data scope investigation]
  M --> L
  L -- yes --> N[Prepare legal and customer notification]
  L -- no --> O[Document no-notification rationale]
  N --> P[Eradicate, recover, monitor]
  O --> P
  P --> Q[Postmortem, action items, control updates]

Important Branches and Edge Cases

  • Alert contains multiple affected accounts, causing dynamic fan-out containment per identity.
  • Containment action fails for a cloud account because break-glass credentials are required.
  • Data exposure cannot be determined immediately, so the workflow holds an SLA deadline while investigation continues.
  • Incident severity changes mid-flight and triggers additional responders and communication obligations.
  • Evidence collection must preserve immutable timestamps and hashes for chain of custody.
  • Postmortem action items must be tracked until completion, not merely created.

Personal Knowledge Archive

Local PC workflow for capturing browser tabs, PDFs, screenshots, notes, transcripts, and searchable summaries.

What It Should Do

Capture information from a user's desktop and browser, normalize it into a local archive, create OCR and summaries, index it for search, sync selected outputs to a notes app, and preserve private data boundaries.

Programs and Services Involved

Capture
Browser extension, clipboard watcher, screenshot tool, email client, PDF folder, audio recorder.
Local Processing
OCR engine, speech-to-text, Markdown converter, local SQLite, vector index, file tags.
Destinations
Obsidian, Logseq, local search UI, encrypted backup, optional cloud summarization API.

Workflow Diagram

flowchart TD
  A[User captures tab, file, screenshot, audio, or clipboard] --> B[Detect content type and privacy label]
  B --> C{Sensitive content?}
  C -- yes --> D[Keep processing local-only]
  C -- no --> E[Allow configured remote enrichments]
  D --> F[Normalize to text, markdown, and original artifact]
  E --> F
  F --> G[Run OCR, transcript, metadata, and link extraction]
  G --> H[Generate title, tags, summary, and backlinks]
  H --> I{User review required?}
  I -- yes --> J[Show inbox item for edit or discard]
  J --> K{Accepted?}
  K -- no --> L[Trash or quarantine]
  K -- yes --> M[Index and write archive note]
  I -- no --> M
  M --> N[Sync selected notes and encrypted backup]

Important Branches and Edge Cases

  • Captured web page has dynamic content, requiring browser-rendered snapshot plus canonical URL metadata.
  • Screenshot contains private information and must not be sent to remote OCR or summarization.
  • Audio transcript takes a long time and should produce partial progress while the item remains searchable by metadata.
  • User edits generated tags before acceptance, and downstream index updates must reflect the edited values.
  • Same URL is captured multiple times, requiring versioning rather than silent overwrite.
  • Encrypted backup fails because the external drive is disconnected, so the workflow waits for the next mount event.

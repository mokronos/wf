Local Media Ingest and Backup

Complex local PC workflow for importing camera cards, deduplicating assets, transcoding proxies, and verifying backups.

What It Should Do

Watch for a camera card or phone mount, copy media to a project library, verify checksums, extract metadata, create editing proxies, update a local catalog, back up to NAS and cloud storage, then safely eject the source only after all required copies are verified.

Programs and Services Involved

Local Tools
udev or launchd, exiftool, ffmpeg, rsync or robocopy, SQLite, desktop notifications.
Storage
Removable SD card, local NVMe library, NAS share, cloud bucket, cold archive disk.
Optional Apps
Lightroom, DaVinci Resolve, PhotoPrism, Syncthing, checksum manifest viewer.

Workflow Diagram

flowchart TD
  A[Detect mounted camera card] --> B[Read volume identity and free space]
  B --> C{Known source?}
  C -- no --> D[Ask user for project, creator, and import policy]
  C -- yes --> E[Load saved import policy]
  D --> F[Scan media and calculate source checksums]
  E --> F
  F --> G{Duplicates found?}
  G -- yes --> H[Skip exact duplicates and flag conflicts]
  G -- no --> I[Copy originals to local library]
  H --> I
  I --> J[Verify copied checksums]
  J --> K{Verification passed?}
  K -- no --> L[Retry copy or quarantine failed files]
  L --> J
  K -- yes --> M[Extract EXIF, GPS, audio, and codec metadata]
  M --> N[Generate thumbnails and proxy files]
  N --> O[Update local catalog database]
  O --> P[Back up to NAS and cloud in parallel]
  P --> Q{All required backups verified?}
  Q -- no --> R[Notify user and keep card mounted]
  Q -- yes --> S[Write import manifest and safely eject card]

Important Branches and Edge Cases

  • Two files have the same name but different hashes, requiring deterministic renaming and conflict reporting.
  • The NAS is offline, so the workflow completes local import but remains pending for backup verification.
  • Large video transcoding should run with limited CPU usage and resume after sleep or reboot.
  • User removes the card early, causing partial import detection and a resumable manifest.
  • Cloud upload hits a quota limit and asks the user whether to skip cloud backup or pause until space is available.
  • GPS metadata may be stripped for privacy when exporting sharing copies while originals remain untouched.
